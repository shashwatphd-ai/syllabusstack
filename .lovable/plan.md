

## Plan: Intelligent Segment-Driven Video Rendering

### Problem
Each slide renders as one static image for its entire duration (60-120s). The narration covers multiple topics (key points, definitions, examples) but the visual never changes — boring and un-watchable.

### Solution
Use the existing `audio_segment_map` (already generated by the audio pipeline) and `parseSegmentMap` fallback to break each slide into **visual scenes**. Each scene gets unique Ken Burns motion, animated text overlays, and cross-fade transitions — all purely additive to the existing working video exporter.

### Architecture — What Changes, What Doesn't

**NOT MODIFIED** (zero risk of breakage):
- `useSlideSync.ts` — imported read-only for `parseSegmentMap`
- `PresentationPlayer.tsx`, `StudentSlideViewer.tsx`, `SlideRenderer.tsx` — untouched
- `LectureSlideViewer.tsx` — untouched
- All database tables, edge functions, storage buckets — untouched
- The `renderLectureVideo` function signature stays identical

**MODIFIED** (additive only):
| File | Change |
|---|---|
| `src/lib/videoExporter.ts` | Extend `VideoSlide` interface with optional `content` + `segmentMap`. Replace single `drawSlideFrame` call per slide with segment-aware loop. Add new drawing functions for text overlays. Existing functions (`drawIntroFrame`, `drawOutroFrame`, `drawCenteredText`) untouched. |
| `src/components/slides/VideoExportButton.tsx` | Pass `content` and `audio_segment_map` through to the exporter when available. Fully backward-compatible — old slides without segment data work exactly as before. |

### How It Works

```text
Slide: "Cell Membrane Structure" (90s, 6 segments)

Segment 1 (0-15s):  main_text     → Full image, title fade-in, Ken Burns zoom-in from center
Segment 2 (15-30s): key_point_0   → Bullet card slides in from right, KB pan left
Segment 3 (30-45s): key_point_1   → Previous bullet dims, new bullet appears, KB zoom top-right
Segment 4 (45-55s): key_point_2   → Third bullet, KB pan right
Segment 5 (55-72s): definition    → Split layout: image left, definition card right, KB zoom center
Segment 6 (72-90s): example       → Example panel overlays bottom half, KB pan up

Each transition: 0.5s cross-fade (15 frames)
```

### Visual Techniques (all topic-agnostic)

1. **Segment-varied Ken Burns** — Each segment gets a unique zoom origin + direction computed from segment index (no two adjacent segments have the same motion)
2. **Text overlay cards** — Semi-transparent dark cards with white text appear for the active content block:
   - `key_point_N`: Bullet card with bullet number badge
   - `definition`: Term + meaning in a bordered card
   - `example`: Scenario text in an accent-bordered panel
   - `step_N`: Step number + title + explanation
   - `main_text`: Title + main text overlay (default)
3. **Fade transitions** — Active card fades in over 15 frames, previous card fades out
4. **Segment counter** — "2 of 5" indicator in top-right corner
5. **Graceful fallback** — If no `segmentMap` or `content` is provided, falls back to current single-image Ken Burns (existing behavior preserved exactly)

### Implementation Details

**`videoExporter.ts` additions:**

New `VideoSlide` fields (all optional for backward compat):
```typescript
content?: {
  main_text?: string;
  key_points?: (string | { text: string })[];
  definition?: { term: string; formal_definition?: string; simple_explanation?: string; meaning?: string };
  example?: { scenario: string; walkthrough?: string; connection_to_concept?: string; explanation?: string };
  steps?: { step: number; title: string; explanation: string }[];
};
segmentMap?: { target_block: string; start_percent: number; end_percent: number }[];
```

New drawing functions:
- `drawSegmentOverlay(ctx, segment, content, fadeProgress)` — Renders the text card for the active segment type
- `drawSegmentKenBurns(ctx, img, segmentIndex, totalSegments, progressInSegment)` — Ken Burns with per-segment variety
- `drawSegmentIndicator(ctx, currentSegment, totalSegments)` — Small "2/5" badge

Modified slide rendering loop (inside `renderLectureVideo`):
- If `slide.segmentMap` exists: iterate through segments, calculate frame ranges per segment, call segment-aware draw functions
- If no segment map: call existing `drawSlideFrame` unchanged

**`VideoExportButton.tsx` additions:**

In the `videoSlides` mapping, add:
```typescript
content: s.content || undefined,
segmentMap: s.audio_segment_map || undefined,
```

Uses `parseSegmentMap` from `useSlideSync.ts` as fallback when `audio_segment_map` is missing but `content` is present.

### No Backend Changes Required

The `audio_segment_map` is already generated by the `generate-lecture-audio` edge function and stored on each slide. The `content` object is already part of ProfessorSlide/EnhancedSlide types. This feature is purely a rendering enhancement that reads existing data.
